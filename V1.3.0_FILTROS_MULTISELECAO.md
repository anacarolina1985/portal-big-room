# v1.3.0 - Guia de Filtros Multi-Seleção

> ⚠️ **Documento histórico** — o componente multi-select descrito aqui (`msPopulate`, `MS_DATA`, valores com separador `|`) foi **substituído na v1.4** pelos dropdowns customizados com busca (`.cac`, single-select), presentes em todos os módulos. Este guia é mantido apenas como registro da versão 1.3.0.

## 🎯 Resumo das Melhorias

A versão 1.3.0 introduz um novo **sistema de múltipla seleção** para os filtros de Vertical, Tribo, Cluster e Squad. Agora você pode selecionar **múltiplos valores simultâneos** em cada filtro, proporcionando uma experiência muito mais flexível e eficiente.

---

## 📊 Como Usar os Novos Filtros

### 1️⃣ Abrindo um Filtro Multi-Seleção
- Clique no botão do filtro desejado (ex: 🌍 Vertical, 👥 Tribo, etc.)
- O dropdown abrirá mostrando todas as opções disponíveis
- Use a **barra de busca** para filtrar opções rapidamente

### 2️⃣ Selecionando Múltiplos Valores
```
Exemplo:
- Clique em "Vertical A" (marca o checkbox ✓)
- Clique em "Vertical B" (marca o checkbox ✓)
- Clique em "Vertical C" (marca o checkbox ✓)
- Resultado: Mostra dados de A, B E C simultaneamente
```

### 3️⃣ Visualizando Seleções
- **Botão do filtro** exibe tags com valores selecionados
- **Barra de status** mostra todos os filtros ativos com ícones
- Exemplo: `🌍 Vertical A, B  |  👥 Tribo X, Y, Z`

### 4️⃣ Limpando Filtros
- Desmarque as opções clicando novamente nos checkboxes
- Ou clique em **"↺ Limpar"** para resetar todos os filtros
- Clique em **"✓ Aplicar"** para executar a filtragem

### 5️⃣ Busca Rápida
- Digite na caixa "Buscar..." do dropdown
- Filtra as opções em tempo real
- Útil para listas grandes com muitas opções

## 🏗️ Arquitetura Técnica

### Estrutura de Dados
```javascript
// Filtros armazenam múltiplos valores separados por "|"
F.v = "Vertical A|Vertical B"     // Múltiplos
F.t = "Tribo X"                   // Um único
F.c = "Cluster A|Cluster B|..."   // Múltiplos
F.s = "Squad 1|Squad 2|..."       // Múltiplos

// Dados do multi-select
MS_DATA = {
  fv: ["Vertical A", "Vertical B", "Vertical C"],
  ft: ["Tribo X", "Tribo Y"],
  fc: ["Cluster A", "Cluster B"],
  fs: ["Squad 1", "Squad 2", "Squad 3"]
}
```

### Componentes CSS Principais
```css
.multi-select        /* Contêiner do componente */
.ms-btn             /* Botão do dropdown */
.ms-val             /* Área de valores selecionados */
.ms-tag             /* Tags dos valores selecionados */
.ms-drop            /* Dropdown/lista de opções */
.ms-opt             /* Opção individual com checkbox */
.ms-search          /* Campo de busca */
.fstatus-item       /* Item na barra de status */
```

### Funções JavaScript

#### `msPopulate(id, opts)`
- Popula as opções do multi-select
- **id**: 'fv', 'ft', 'fc', 'fs'
- **opts**: Array de opções

#### `msToggle(id)`
- Abre/fecha o dropdown
- Gerencia exclusão mútua (só um dropdown aberto)

#### `msToggleOpt(id, val)`
- Marca/desmarca uma opção
- Atualiza `F[campo]` com separador `|`

#### `msUpdateDisplay(id)`
- Atualiza visualização do botão e status
- Renderiza as tags de seleção

#### `msFilter(id)`
- Filtra opções pela busca
- Mostra/oculta itens em tempo real

#### `filtered()`
- Filtra dados baseado em múltiplas seleções
- Suporta lógica OR (matches ANY selected value)

---

## 🔄 Fluxo de Filtragem

```
User clica opção
       ↓
msToggleOpt() → atualiza F[campo] com "|"
       ↓
msUpdateDisplay() → renderiza tags no botão
       ↓
doFilter() → executa filtered()
       ↓
updateFStatus() → mostra barra de status
       ↓
render() → exibe dados filtrados
```

## 🚀 Performance

- **Busca**: O(n) onde n = número de opções
- **Toggle**: O(1) para add/remove
- **Display Update**: O(n) renderização
- **Filtragem**: O(n*m) onde n = dados, m = filtros ativos

**Otimizações:**
- Debounce de 120ms no `onAC()` para aplicação
- Event delegation para close-on-outside
- Array.indexOf() para busca de valores

## ⚠️ Considerações de Compatibilidade

### Breaking Changes
- Formato de valores mudou de `"Valor"` para `"Valor1|Valor2"`
- Função `popDL()` mantida apenas para Quarter (não multi-select)

### Backward Compatibility
- Dados antigos com um único valor ainda funcionam
- Sem mudanças no CSV parsing
- Funções antigas preservadas quando possível
